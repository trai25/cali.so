# UI on shadcn + Tailwind v4, extended from the @fluid registry

v3's component layer is shadcn/ui on Tailwind v4 (CSS-variable tokens; the legacy v2 Tailwind v3 config dies with the old code), seeded from the `@fluid` registry (fluidfunctionalism.com) for spring-physics, motion-as-information components, and extended with our own components in the same idiom. Custom components follow the registry's conventions (tokens, motion principles, accessibility defaults) rather than inventing a parallel style, so the two sources stay indistinguishable in the UI.
